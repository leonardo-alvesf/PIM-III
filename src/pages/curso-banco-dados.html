SQL vs NoSQL

Entenda as diferenças entre bancos relacionais e não-relacionais.

Bancos SQL (Relacionais)

Características:

  • Estrutura de tabelas com linhas e colunas
  • Schema rígido e predefinido
  • Relacionamentos entre tabelas
  • ACID (Atomicidade, Consistência, Isolamento, Durabilidade)

Quando usar:

  • Dados estruturados e relacionados
  • Necessidade de transações complexas
  • Integridade referencial importante
  • Queries complexas com JOINs

Exemplos:

MySQL · PostgreSQL · SQL Server · Oracle

Bancos NoSQL (Não-Relacionais)

Características:

  • Estrutura flexível
  • Schema dinâmico
  • Alta escalabilidade horizontal
  • BASE (Basically Available, Soft state, Eventually consistent)

Tipos:

  • Documento: MongoDB, CouchDB
  • Chave-Valor: Redis, DynamoDB
  • Coluna: Cassandra, HBase
  • Grafo: Neo4j, ArangoDB

Quando usar:

  • Grandes volumes de dados
  • Necessidade de alta escalabilidade
  • Dados não estruturados ou semi-estruturados
  • Desenvolvimento ágil com schema flexível

Comparação:

Schema

/* SQL — schema fixo */ CREATE TABLE usuarios ( id INT PRIMARY KEY, nome VARCHAR(100) NOT NULL, email VARCHAR(200) UNIQUE ); /* NoSQL (MongoDB) — schema livre */ db.usuarios.insertOne({ nome: "Ana", email: "ana@email.com", perfil: { nivel: "admin" } // campo extra sem migração });

Escalabilidade

  • SQL: escalabilidade vertical (mais CPU/RAM no mesmo servidor)
  • NoSQL: escalabilidade horizontal (mais servidores em paralelo)

Consistência vs Disponibilidade

  • SQL/ACID: prioriza consistência — dados sempre corretos
  • NoSQL/BASE: prioriza disponibilidade — eventualmente consistente
